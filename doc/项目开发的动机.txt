# 🚀 项目代号：AI PPT Agent (Bento-SVG Edition)
## 最终版项目背景与核心业务流转说明书 (PRD & Context)

### 一、 项目愿景与痛点分析 (Project Vision & Background)
当前市面上的“一键生成 AI PPT”工具普遍存在致命缺陷：它们跳过了“内容与逻辑编排”阶段，仅根据用户输入的简单主题，套用粗糙的大纲和僵硬的预设模板，导致产出的 PPT 缺乏灵魂，甚至在面对用户自带的复杂业务文档（如财报、企划书）时直接瘫痪。

**本项目的核心愿景**：彻底颠覆“一键套模板”模式，在数字世界中复刻**“顶尖商业 PPT 定制公司（报价 1万+/页）的人类专家工作流”**。让 AI 分别扮演业务顾问、提炼专家、策划师和设计师。以“内容与金字塔逻辑”为核心驱动，结合“便当网格（Bento Grid）”排版算法，最终输出具备极高商业交付价值的原生 SVG 设计稿。

### 二、 核心架构原则：后端驱动工作流 (Backend-Driven Orchestration)
为了保护核心资产、控制大模型 API 成本并确保长链路任务的稳定性，本项目的**核心提示词组装、API 调度、外部检索与数据清洗（JSON/SVG 解析）必须 100% 在后端（Backend）完成**。
前端（Frontend）仅负责收集需求、渲染“数字便利贴”进行大纲交互，以及最终 SVG 的可视化预览。

### 三、 核心业务流转管道 (The 4-Step Pipeline)

#### 阶段 1：智能输入网关 (The Ingestion Gateway) - 【双轨并跑】
系统入口需具备意图识别能力，提供双轨并行的工作流：
* **轨道 A：启发提问模式（Zero to One）**：用户只有模糊主题。AI 调用搜索引擎（如 Grok 或国内平替）获取市场现状，化身“业务顾问”向用户提问，明确受众与目标。
* **轨道 B：文档解析模式（Doc to PPT）**：用户上传现成材‍料（长文本/Word/PDF）。系统直接提取文本，AI 化身“阅读理解专家”，进入强约束的无损压缩与提炼状态，严防幻觉。

#### 阶段 2：大纲重构与逻辑梳理 (The Architect)
无论走哪条轨道，数据最终汇聚于此。AI 运用“金字塔原理”，将信息重构为严格的 JSON 大纲。在前端，这个 JSON 会被渲染成一张张“数字便利贴”，用户可直观地拖拽、增删和调换页面逻辑。

#### 阶段 3：策划与排版骨架 (The Layout Planner)
确认大纲后，系统进入策划阶段。底层引擎为每个页面节点分配“便当网格（Bento Grid）”版式。卡片式布局灵活性极强，且其本质是二维坐标系的区块划分，是大模型最容易理解的高级设计语言。

#### 阶段 4：矢量设计与代码渲染 (The SVG Renderer)
放弃 HTML 和传统模板库，模型直接结合具体文案与排版骨架，生成整页 SVG 代码。
* **商业价值**：SVG 兼容 Office 2016+，用户导出并拖入 PPT 后，可直接“取消组合”进行二次细节编辑，完美契合真实办公刚需。

#### 💡 Prompt 1-A：启发式结构架构师 (对应轨道 A：从 0 到 1)
```markdown
# Role: 顶级的PPT结构架构师 (Zero-to-One)

## Profile
- 版本：2.0 (Context-Aware)
- 专业：PPT逻辑结构设计
- 特长：运用金字塔原理，结合**背景调研信息**构建清晰的演示逻辑

## Goals
基于用户提供的 **PPT主题** 和 **背景调研信息 (Context)**，设计一份逻辑严密、层次清晰的PPT大纲。

## Core Methodology: 金字塔原理
1. 结论先行：每个部分以核心观点开篇
2. 以上统下：上层观点是下层内容的总结
3. 归类分组：同一层级的内容属于同一逻辑范畴
4. 逻辑递进：内容按照某种逻辑顺序展开

## 重要：利用调研信息
你将获得一些关于主题的搜索摘要。请务必参考这些信息来规划大纲，使其切合当前的市场现状或技术事实，而不是凭空捏造。

## 输出规范
请严格按照以下JSON格式输出，结果用[PPT_OUTLINE]和[/PPT_OUTLINE]包裹：
[PPT_OUTLINE]
{
  "ppt_outline": {
    "cover": { "title": "主标题", "sub_title": "副标题", "content": [] },
    "table_of_contents": { "title": "目录", "content": ["部分1", "部分2"] },
    "parts": [
      {
        "part_title": "第一部分：章节标题",
        "pages": [ { "title": "页面标题1", "content": [] } ]
      }
    ],
    "end_page": { "title": "总结与展望", "content": [] }
  }
}
[/PPT_OUTLINE]
```

#### 💡 Prompt 2：Bento Grid 排版方法论 (版式核心算法)
*(注：这是一个独立的排版逻辑模块，在生成阶段作为 Context 拼接到 Prompt 3 中)*
```markdown
内容页的便当网格 (Bento Grid) 布局
这是一种灵活的网格系统，其布局应由内容本身的需求驱动，而非僵硬的模板。通过组合不同尺寸的卡片，创造出动态且视觉有趣的布局。

- 核心原则:
    - 灵活性: 卡片数量不固定。可以是 1, 2, 3, 4, 5 或更多个。
    - 层级感: 使用卡片尺寸建立视觉层级。最重要的信息放在最大的卡片上。
    - 留白: 在所有卡片之间保持至少 20px 的间距。

- 布局组合示例:
    - 单一焦点: 一张大卡片覆盖大部分区域 (w=1200, h=580)。
    - 两栏布局: 50/50 对称两张卡片，或非对称(2/3 与 1/3)。
    - 三栏布局: 三张等宽的卡片。
    - 主次结合: 居中大卡片，两侧各一张小垂直卡片。
    - 顶部英雄式: 顶部宽幅卡片，下方 2-4 个小等宽卡片网格。
    - 混合网格 (自由度最高): 自由混合，如一中等方块+两水平矩形+一垂直矩形。
```
